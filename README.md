# Poiyatsu_movie

　■ サービス概要

    3行で
      エンタメの作り手になりたいという潜在的欲求を持っている人に
      誰でも簡単に誰かに見せてみたくなるような動画を作れる
      エンタメ動画の作成アプリです

　■メインのターゲットユーザー

  youtubeチャンネル「アニメOPっポイヤツ」の視聴者
  アニメ、漫画が好きな人
  お芝居（観る、演じる）や、ちょっと変わったエンタメが好きな人

　■ユーザーが抱える課題・欲求

    youtubeチャンネル「アニメOPっポイヤツ」の視聴者コメントの特徴として、「参加してみたい、仲間と楽しそうで羨ましい」というコメントや、自分なりのストーリーを創作して上げているコメントが多い。

    →エンタメを受け取る側ではなく、作る側になりたい、そして、それを誰かに見てもらいたい、共有したいという潜在的欲求がある。
    しかし、0から作り手になるには様々なハードルがあり、簡単にチャレンジするには難しいという現状がある。

  ハードル例

    ・一人で作るのが難しい。例えば、芝居、漫才、コントなどの場合、基本的には一人では成立しない。カメラマンの存在も欲しくなる。
    ・一人でも成立するエンタメは、クオリティ勝負になりやすい分野が多く、発信ハードルが高い。(歌、演奏、ダンス、絵）自分くらいのレベルでわざわざ公開するのもな、、という状態になりやすい。
    ・総じてエンタメは、見せるものとして成立するまでに必要とされる手間や努力が多い。
    （また、誰でも評価しやすい分野なので、ネガティブなコメントに対する恐れもある。）

　■解決方法

    ユーザーが投稿してくれた動画（簡単なアクションをする動画）と、アクター（パントマイムの人）が演技している動画を合成して、ストーリーを成立させることで、一緒に演技しているようなエンタメ動画を作成する。
    起承転結の、起をユーザーに担ってもらい、承転結をこちら側で担うことで、ユーザーの手間はかからずとも、「面白い動画を自分で作れた！制作に参加できた！」と思ってもらえることを目指します。

  ※動画を撮影するというアクションをユーザーに求めるため、スマートフォンを想定しています。

    ・動画作成を自分も試してみようと思ったユーザー

      ・ユーザーが手軽に簡単に合成動画を作れる。
        ・ユーザーがコンテンツ一覧画面から、撮ってみたいコンテンツを選択できる。（サムネイルと説明文で、そのコンテンツがどんなネタか少し想像できる）
        ・ユーザーが撮った動画をアップロードできる。

# **テーブル詳細**

## アプリ趣旨：

アプリ製作者が用意した動画（content）があり、ユーザーにはその中から面白そうなものをチョイスしてもらいます。

各コンテンツには「ユーザー自身が撮って投稿してほしい数秒の動画」というものが指定されています。それを撮って投稿してもらった後、バックエンド（AWS lambda + FFmpeg）で投稿動画（users_video）とコンテンツ動画を加工、合成します。

その結果、一連のストーリーが生まれる完成動画（completed_video)が出来上がり、それを視聴して楽しんでもらう流れになります。

## **users**

ユーザー情報を管理します。

- name: ユーザー名
- email: ユーザーアドレス
- avatar: ユーザーアイコン(googleアカウントのイメージ）
- role(enum): ユーザーのステータス。非会員、会員、管理者の３つの分類想定
- crypted_password: sorceryによってハッシュ化されるパスワード
- salt: sorceryによってパスワードに付加される文字列

※ 認証まわりは、十分に検討、検証ができておらず、その後変更するかもしれません。

## users_videos

ユーザーが、選択したコンテンツの指示に沿って撮影・投稿した動画を管理します。

 FFmpegを使用して、「mp4に変換」を行い、AWS 3Sにアップロードします。

- number: ユーザー動画に割り振る一意の番号

３S、lambda関数と連携を取る際に、使い勝手のいい一意のデータが欲しくなる可能性が高いので、idとは別に用意しています。

- vide_url: 投稿動画の一時保管場所
- transcoded_video_url: mp4変換後の投稿動画の一時保管場所
- content_number: 選択したコンテンツの番号

## contents

コンテンツ動画を管理します。

- number: コンテンツ動画に割り振る一意の番号

３S、lambda関数と連携を取る際に、使い勝手のいい一意のデータが必要になるため、idとは別に用意しています。

- video_url:　コンテンツ動画の保管場所
- title: コンテンツ動画のタイトル（キャッチコピーのイメージ）
- description: コンテンツ動画の説明（概要欄のイメージ）
- category: コンテンツのジャンル（アクション編・パントマイム編・恋愛編 etc.)
- video_length: 動画の尺

動画の長さで、加工の種類や動画コンセプトが区分けされてくることが想定されるために用意する予定です。例えば、長尺動画は会員登録をした人のみ公開する、など。

長尺動画はコンテンツとしてのボリュームが高まり満足度は上がりますが、その一方で、加工完了までの待ち時間のストレスは増えるため、ある程度エンゲージメントが高まった人に限定した方がいいかな？など考えています。

- published_at: 公開日（予定含め）　公開日を事前告知する動画用
- deleted_at: 削除日（予定含め）　期間限定公開の動画用

## completed_videos

ユーザーが完成させた動画を管理します。

完成させた動画はサイト内での公開、非公開を選ぶことができます。公開された場合、他ユーザーは、コメントやいいね（今回は、ポイね！と表記）をすることができます。

- number: 完成動画に割り振る一意の番号

３S、lambda関数と連携を取る際に、使い勝手のいい一意のデータが欲しくなる可能性が高いので、idとは別に用意しています。

- video_url:　完成動画の保管場所
- user_title: ユーザー自身がつけれるタイトル

## comments

completed_video(ユーザーの完成動画)のコメント情報を管理します。

## lik**es**

completed_video(ユーザーの完成動画)のポイね！情報を管理します。（いいねのオマージュ）

## love**s**

completed_video(ユーザーの完成動画)の推し！情報を管理します。（お気に入りのオマージュ）
